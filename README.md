# CS218 — MerkleMinds DAO

Project 3 — **DAO Governance Contract** for CS 218 (Programmable & Interoperable Blockchain).

A decentralised governance system implementing ERC-20 token-weighted voting with
snapshot-based vote weights, 30 % quorum enforcement, time-windowed voting,
on-chain proposal execution against a Treasury contract, admin-controlled
quorum parameter, and proposer-only cancellation during the pending phase.

## Contract Overview — `DAO.sol`

| Function | Caller | Purpose |
| -------- | ------ | ------- |
| `constructor(token, votingDelay)` | deployer | sets ERC-20Votes token + voting delay |
| `createProposal(desc, target, data, votingPeriod)` | any token holder | records snapshot block at creation |
| `vote(proposalId, support)` | any token holder | one vote per address, weight = past votes at snapshot |
| `executeProposal(proposalId)` | anyone | after deadline, if quorum met + majority yes, executes `target.call(data)` |
| `cancelProposal(proposalId)` | proposer only | only during Pending phase (before voting starts) |
| `setQuorumPercent(p)` | `ADMIN_ROLE` | admin updates quorum percent |
| `getProposal(id)` / `getProposalStatus(id)` | view | status, votes, quorum reached |

Status enum: `Pending | Active | Defeated | Succeeded | Executed | Cancelled`.

## Security Notes

- `ReentrancyGuard` on `executeProposal`, `Treasury.release`, `Treasury.transferETH`.
- Snapshot vote weight via `ERC20Votes.getPastVotes(voter, snapshotBlock)` — kills
  flash-loan / last-minute buy-in attacks.
- `hasVoted[proposalId][voter]` prevents double voting.
- Quorum: `(yesVotes + noVotes) * 100 >= totalSupply * quorumPercent` (30 %).
- `AccessControl` gates `setQuorumPercent` (deployer gets `ADMIN_ROLE` + `DEFAULT_ADMIN_ROLE`).
- Checks-effects-interactions: `executed = true` is set before the external call.
- `executeProposal` reverts if the downstream call reverts.
